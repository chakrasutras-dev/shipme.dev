# ShipMe Product Specification v1.0

> AI-Powered Infrastructure Provisioning Platform

**Version:** 1.0
**Date:** February 2026
**Status:** Implemented (Initial Release)

---

## Executive Summary

ShipMe is a web-based platform that helps developers provision cloud infrastructure using natural language. Users describe their project, and ShipMe generates a detailed infrastructure plan with step-by-step guidance to set up their stack.

---

## Product Vision

**Problem:** Setting up modern cloud infrastructure is complex and time-consuming. Developers must navigate multiple dashboards (GitHub, Supabase, Vercel/Netlify, Google Cloud) and manually configure authentication, databases, and deployments.

**Solution:** ShipMe provides an AI-powered assistant that understands project requirements and generates comprehensive infrastructure plans with guided setup instructions.

## Core Features

### 1. User Authentication
- Google OAuth via Supabase Auth
- GitHub OAuth via Supabase Auth
- Session management
- User profiles stored in Supabase

### 2. Project Intake
- Natural language project description
- AI-powered clarifying questions
- Project type detection (SaaS, API, mobile backend, etc.)

### 3. Infrastructure Planning
- AI generates comprehensive infrastructure plan
- Recommended technology stack
- Database schema suggestions
- Authentication flow recommendations
- Deployment strategy

### 4. Guided Setup Instructions
- Step-by-step instructions for each service
- Screenshots and documentation links
- Code snippets for configuration
- Environment variable templates

### 5. Admin Dashboard
- User management (admin role)
- Service integrations management
- Activity logging
- Analytics

## User Journey

```
1. LANDING PAGE
   User visits shipme.dev
   Sees value proposition
   Clicks "Get Started"
           │
           ▼
2. AUTHENTICATION
   User signs in with Google or GitHub
   Profile created in Supabase
           │
           ▼
3. PROJECT DESCRIPTION
   User describes their project in chat
   "I want to build a marketplace for handmade goods..."
           │
           ▼
4. AI CONVERSATION
   Claude asks clarifying questions
   - "Do you need user authentication?"
   - "Will you process payments?"
   - "What's your expected scale?"
           │
           ▼
5. PLAN GENERATION
   Claude generates infrastructure plan:
   - Recommended: Supabase + Netlify + Stripe
   - Database schema
   - Auth configuration
   - Deployment steps
           │
           ▼
6. MANUAL SETUP (User does this themselves)
   User follows step-by-step guide:
   □ Create Supabase project
   □ Run database migrations
   □ Configure Google OAuth in GCP
   □ Add OAuth credentials to Supabase
   □ Deploy to Netlify
   □ Set environment variables
   □ Configure custom domain
           │
           ▼
7. COMPLETION
   User has working infrastructure
   (30-45 minutes of manual work)
```

## Database Schema

### Tables

**profiles**
```sql
- id (uuid, PK, references auth.users)
- email (text)
- full_name (text)
- avatar_url (text)
- role (text: 'user' | 'admin')
- created_at (timestamp)
- updated_at (timestamp)
```

**automations**
```sql
- id (uuid, PK)
- user_id (uuid, FK → profiles)
- title (text)
- description (text)
- status (text: 'planning' | 'ready' | 'provisioning' | 'completed' | 'failed')
- project_type (text)
- created_at (timestamp)
- updated_at (timestamp)
```

**automation_plans**
```sql
- id (uuid, PK)
- automation_id (uuid, FK → automations)
- plan_data (jsonb) -- Full infrastructure plan
- version (integer)
- created_at (timestamp)
```

**service_integrations** (Admin)
```sql
- id (uuid, PK)
- name (text)
- description (text)
- category (text)
- logo_url (text)
- is_enabled (boolean)
- is_coming_soon (boolean)
- api_spec (jsonb)
- provisioning_steps (jsonb)
```

## Limitations of v1.0

### Manual Credential Handling
- Users must copy/paste credentials between services
- Error-prone process
- Security concerns (credentials in clipboard)

### No Actual Provisioning
- ShipMe only provides instructions
- User does all the actual work
- No verification that steps were completed correctly

### Time-Consuming
- 30-45 minutes of manual setup
- Multiple dashboard logins required
- Context switching between services

### No Visibility for Non-Technical Users
- Assumes familiarity with cloud dashboards
- No hand-holding through complex configurations
- Easy to get lost or make mistakes

### Credential Security
- No secure way to handle secrets
- Users might store credentials insecurely
- No audit trail of credential usage

## Deployment

### Prerequisites
- GitHub account
- Supabase account
- Google Cloud account (for OAuth)
- Netlify account
- Anthropic API key

### Environment Variables
```
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...
ANTHROPIC_API_KEY=sk-ant-...
```

### Deployment Steps
1. Push code to GitHub
2. Create Supabase project, run migrations
3. Configure Google OAuth in GCP
4. Connect Google OAuth to Supabase
5. Deploy to Netlify
6. Set environment variables
7. Configure custom domain

See: [DEPLOYMENT_GUIDE.md](../DEPLOYMENT_GUIDE.md)
